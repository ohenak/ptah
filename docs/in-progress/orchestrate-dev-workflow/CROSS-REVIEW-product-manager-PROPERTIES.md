## Recommendation

**Need Attention**

> F-01, F-02, and F-03 are High findings. Three product-visible behaviors — the `revision_bound` validation error path, the YAML validator's SkipCondition type acceptance, and the phase transition output lines — have no property coverage. These represent gaps where implementation could deviate from approved acceptance criteria without any test catching it. Medium findings F-04 through F-07 should also be addressed before sign-off. Specifically:
>
> - Add a property for the REQ-WF-04 validation-error path (a custom YAML with a review phase missing `revision_bound` → `ptah run` exits 1 with a named-phase error).
> - Add properties for the YAML validator accepting the `artifact.exists` SkipCondition and rejecting malformed variants (missing `artifact` field; spurious `equals` field).
> - Extend PROP-CLI-18 or add a new property to explicitly cover phase transition output lines (`Passed ✅`, `Revision Bound Reached ⚠️`) and the optimizer-dispatch line.
> - Revisit PROP-INIT-02's hardcoded count of 20 and either confirm it from the manifest or replace it with a derivation rule.
> - Add a higher-level Integration property for REQ-NF-01 verifying `ptah start` continues to work post-upgrade.
> - Remove or reclassify PROP-CNP-02's deep-copy requirement as a TSPEC concern; adjust the REQ citation accordingly.
